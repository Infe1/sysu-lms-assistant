# 中山大学 LMS 学习助手 LLM 答题版

> 一个运行在 Tampermonkey 上的用户脚本，用于辅助完成中山大学 LMS 课程中的视频学习、章节测验与页面推进流程。

## 项目来源

本项目基于原项目 [`ntgmc/sysu-lms-assistant`](https://github.com/ntgmc/sysu-lms-assistant) 修改而来。

在原项目基础上，本版本进行了较大幅度的功能扩展，重点加入了：

- 基于 **OpenAI 兼容接口** 的 LLM 自动答题能力
- 多阶段测验自动处理与自动提交
- 视频完成后的自动刷新与“下一活动”推进逻辑
- 反焦点检测 / 后台保活增强
- 可视化设置面板
- 运行日志、手动清空日志、按天自动清空日志

## 功能概览

### 自动学习流程

脚本会在 LMS 页面中自动处理以下流程：

1. 检测课程视频页面
2. 自动尝试播放视频
3. 自动切换为“超清”画质
4. 视频被意外暂停时自动恢复播放
5. 视频完成后等待一小段时间刷新页面
6. 刷新后按 LMS 原生“下一活动”逻辑进入下一项内容
7. 若下一活动为章节测验，则自动进入测验流程
8. 自动答题、自动提交、自动检查得分
9. 满分或达到重试上限后继续进入下一节

### 支持的测验流程

当前脚本支持 Moodle / LMS 中较常见的如下流程页：

- `view.php`：测验概览页
- `attempt.php`：答题页
- `summary.php`：提交前概要页
- `review.php`：查看得分页

### 当前支持的题型

- 判断题
- 单选题
- 多选题

> 说明：不保证支持所有特殊题型，例如填空题、拖拽题、排序题、主观题、复杂材料题等。

### LLM 接口支持

脚本使用 **OpenAI 风格 `/chat/completions` 接口**，因此理论上兼容 OpenAI Chat Completions 协议的服务

### 额外辅助能力

- 左侧悬浮按钮控制运行/暂停
- 可视化填写 `Base URL / Model / API Key`
- 讨论页自动跳过
- 反焦点检测开关
- 日志下载
- 日志手动清空
- 日志按天自动清空

## 使用方法

### 1. 安装 Tampermonkey

先在浏览器中安装 Tampermonkey 扩展。

### 2. 导入脚本

将 `script.js` 导入 Tampermonkey，或新建脚本后将文件内容完整复制进去并保存。

### 3. 打开 LMS 并手动登录

进入中山大学 LMS，先完成登录与必要授权。

> 脚本不会替你处理统一认证登录。

### 4. 进入课程页面

进入具体课程内容页面后，脚本即可开始工作。

## 使用方法

### 第一步：配置 LLM

脚本默认从浏览器 `localStorage` 中读取 LLM 配置。

你可以通过两种方式配置：

#### 方式 A：页面左侧设置按钮

打开 LMS 页面后，点击左侧 `⚙️` 按钮，填写：

- API Base URL
- Model
- API Key
- 是否启用反焦点检测

点击保存即可。

#### 方式 B：浏览器控制台手动配置

可在控制台执行类似命令：

- `localStorage.setItem('lms_llm_base_url', 'https://api.deepseek.com/v1')`
- `localStorage.setItem('lms_llm_model', 'deepseek-chat')`
- `localStorage.setItem('lms_llm_api_key', 'sk-xxxx')`

配置完成后刷新 LMS 页面。

### 第二步：开始自动学习

点开第一个视频后，脚本启动后会自动执行：

- 自动播放视频
- 自动切换超清
- 视频完成后自动刷新页面
- 继续进入“下一活动”
- 如下一活动为测验，则自动答题并提交

### 第三步：使用左侧悬浮按钮

脚本会在页面左侧注入若干控制按钮：

- `LMS助手: 运行中 / 已暂停`：总开关
- `⚙️`：配置 LLM 接口
- `🛡️ / 🔓`：开启/关闭反焦点检测
- `📥`：下载日志
- `🧹`：清空日志

## LLM 配置说明

```js
const LLM_CONFIG = {
  base_url: 'https://api.deepseek.com/',
  model: 'deepseek-v4-flash',
  api_key: 'your_api_key',
  max_tokens: 1024,
  temperature: 0.1,
};
```

## 注意事项

1. **请先手动登录 LMS**，脚本不会自动绕过认证流程。
2. **请先配置 LLM API Key**，否则章节测验无法自动答题。
3. LLM 回答不保证始终正确，因此不保证每次都能直接满分。
4. 当前脚本适配的是当前版本下的 LMS 页面结构；一旦 LMS 前端改版，脚本可能失效。
5. 不同浏览器对后台标签页的限制不同，因此反焦点检测与后台保活不保证 100% 有效。
6. `API Key` 会保存在浏览器本地存储中，请自行注意本机安全。
7. 本脚本仅适用于你本人已获得访问权限的课程页面。

## 常见问题

### 为什么脚本没有自动运行？

请检查：

- Tampermonkey 脚本是否已启用
- 当前页面是否在 `https://lms.sysu.edu.cn/` 域名下
- 是否已成功登录 LMS
- 是否打开一个课程视频页面
- 左侧总开关是否处于“运行中”

### 为什么测验没有自动作答？

请检查：

- 是否已配置 `API Key`
- `Base URL` 是否正确
- 模型名是否正确
- 当前平台是否兼容 OpenAI Chat Completions 接口

### 为什么视频完成后脚本会刷新页面？

因为 LMS 有时不会立即更新当前章节完成状态与“下一活动”目标。刷新页面后，平台更容易正确暴露下一步内容，从而减少跳转错误。

### 为什么会自动重考？

当前版本支持章节测验在未满分时自动重试，直到达到重试上限或拿到满分。（仍需优化）

## 适用范围与限制

本项目是一个**课程页面自动化脚本**，不是官方产品，也不是通用题库系统。

它依赖于：

- 当前 LMS 的页面结构
- 当前课程的活动组织方式
- 当前题型结构
- 所接入 LLM 的回答质量

因此你应当预期：

- 某些课程可用，某些课程不可用
- 某些章节稳定，某些章节不稳定
- 某些题型支持，某些题型不支持

如需进一步适配，请自行修改 `script.js`。

## 开发与二次修改建议

如果你计划继续修改，建议优先关注以下部分：

- `LLM_CONFIG`：模型配置
- `handleQuiz()`：测验主流程
- `extractQuestions()`：题目提取逻辑
- `parseLLMResponse()`：LLM 结果解析逻辑
- 主循环 `mainLoop`：视频、刷新、跳转逻辑

建议在修改时保留：

- 详细日志
- 多策略按钮查找逻辑
- 页面结构变动的容错处理
